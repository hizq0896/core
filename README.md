# core

> 斗鱼入门到进阶的学习路线与资源汇总，帮助学习者建立清晰认知、系统积累方法，并逐步形成可落地的实践能力。🚀

## 简介

`core` 是一个面向斗鱼学习者整理的开源资料仓库，聚焦“从入门到进阶”的成长路径设计。无论你是刚开始接触斗鱼相关内容，希望快速建立基础框架；还是已经有一定经验，想进一步完善知识体系、寻找实践方向，这个项目都希望为你提供一份清晰、可持续更新的参考地图。

项目内容以“路线 + 资源 + 方法”为核心：  
- **路线**：按阶段划分学习目标，减少盲目搜索与重复试错；  
- **资源**：筛选优质文档、社区内容与常用工具，降低信息噪音；  
- **方法**：强调从了解、实践到复盘的闭环，帮助你把零散信息转化为实际能力。  

相比单纯的链接收藏夹，`core` 更关注学习过程中的“节奏感”。你可以把它当成一个知识导航站，用于梳理方向、补齐短板、记录阶段性成果，并逐步构建自己的斗鱼学习体系。📚

## 特点

- **分阶段学习路线**：覆盖入门、熟悉、进阶等不同阶段，目标更明确。
- **资源聚合更高效**：集中整理常用资料，减少来回检索的时间成本。
- **兼顾理论与实践**：不仅关注“看什么”，也强调“怎么练”与“如何验证”。
- **适合自学与团队共建**：个人可按路线推进，团队也可基于仓库持续扩展。
- **结构清晰，便于维护**：适合通过 PR、Issue 持续补充经验和最新资料。
- **关注长期积累**：鼓励记录笔记、案例、踩坑经验，形成可复用知识库。
- **开放协作友好**：欢迎不同背景的学习者贡献内容，共同完善路线图。✨

## 快速开始

如果你希望快速使用本仓库，建议按以下方式开始：

### 1. 克隆项目

```bash
git clone https://github.com/your-username/core.git
cd core
```

### 2. 浏览目录

建议优先查看以下内容（可按你的实际仓库结构调整）：

```text
core/
├── README.md
├── docs/
│   ├── beginner.md
│   ├── intermediate.md
│   └── advanced.md
├── resources/
│   ├── tools.md
│   ├── links.md
│   └── faq.md
└── examples/
```

### 3. 选择学习路径

你可以根据自己的当前阶段切入：

- **入门阶段**：先了解基本概念、核心背景、常见术语。
- **熟悉阶段**：开始系统阅读资料，建立自己的知识框架。
- **进阶阶段**：结合案例、工具与实际问题，开展专题学习与输出。

### 4. 建议的学习方式

- 每周固定一个主题进行学习；
- 为每个主题补充 1 份笔记；
- 将遇到的问题记录到 `faq` 或个人文档中；
- 学完一个阶段后进行复盘，更新自己的知识地图。

## 使用说明

本仓库更适合作为“学习导航”和“资料总表”来使用，而不是一次性读完的文档集合。

### 学习建议

1. **先搭框架，再补细节**  
   不建议上来就深挖单一话题。先建立整体认知，明确有哪些模块、它们之间是什么关系，再逐步展开。

2. **重视输出**  
   仅仅收藏资源往往难以真正掌握。建议把每次学习沉淀成简短笔记、问题清单或实践记录，这样更容易形成长期记忆。

3. **建立自己的版本**  
   `core` 提供的是公共参考路线，但每个人的背景、时间和目标都不同。你可以基于本项目二次整理，形成更适合自己的学习计划。

4. **通过 Issue / PR 参与共建**  
   如果你发现某些资源失效、内容过时，或有更好的学习材料，欢迎通过 Issue 提出建议，或直接提交 Pull Request。

### 适合人群

- 想系统了解斗鱼相关内容的初学者
- 已有一定基础，希望构建完整知识体系的学习者
- 希望沉淀团队内部学习资料的协作者
- 想通过公开整理资料来促进自我成长的开源参与者

### 贡献方式

欢迎补充以下类型内容：

- 学习路线优化建议
- 优质文章、工具、课程与文档
- 实践经验、案例分析、避坑指南
- 错别字修正、格式优化、失效链接更新

可参考如下流程：

```bash
fork 本仓库
git checkout -b feat/update-resources
# 修改内容
git commit -m "docs: update learning resources"
git push origin feat/update-resources
# 提交 Pull Request
```
